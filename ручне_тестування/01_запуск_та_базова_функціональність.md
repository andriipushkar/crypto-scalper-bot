# Тести запуску та базової функціональності

## Підготовка до тестування
- [ ] Переконатися, що віртуальне середовище активоване: `source .venv/bin/activate`
- [ ] Переконатися, що всі залежності встановлені: `pip install -r requirements.txt`
- [ ] Скопіювати `.env.example` в `.env` та налаштувати

---

## 1. Запуск сервера

### TC-001: Запуск FastAPI сервера
**Кроки:**
1. Виконати команду: `uvicorn src.web.app:app --reload --port 8000`
2. Дочекатися повідомлення про успішний запуск

**Очікуваний результат:**
- [ ] Сервер запустився без помилок
- [ ] В консолі відображається: `Uvicorn running on http://127.0.0.1:8000`
- [ ] Немає помилок імпорту модулів

### TC-002: Доступ до головної сторінки
**Кроки:**
1. Відкрити браузер
2. Перейти на `http://localhost:8000`

**Очікуваний результат:**
- [ ] Сторінка завантажилась
- [ ] Відображається дашборд
- [ ] Немає JavaScript помилок в консолі браузера (F12 -> Console)

### TC-003: Перевірка health endpoint
**Кроки:**
1. Виконати: `curl http://localhost:8000/api/health`

**Очікуваний результат:**
- [ ] Відповідь 200 OK
- [ ] JSON з полями status, uptime

## 4. WebSocket з'єднання

### TC-011: Встановлення WebSocket з'єднання
**Кроки:**
1. Відкрити DevTools -> Network -> WS
2. Перезавантажити сторінку

**Очікуваний результат:**
- [ ] WebSocket з'єднання встановлене
- [ ] URL: ws://localhost:8000/ws
- [ ] Статус: Connected (101)

### TC-012: Отримання оновлень через WebSocket
**Кроки:**
1. Спостерігати за повідомленнями в WS

**Очікуваний результат:**
- [ ] Приходять повідомлення типу: status, prices, positions, metrics
- [ ] Дані оновлюються на UI в реальному часі

### TC-013: Перепідключення при втраті з'єднання
**Кроки:**
1. Зупинити сервер (Ctrl+C)
2. Запустити сервер знову
3. Почекати 5-10 секунд

**Очікуваний результат:**
- [ ] WebSocket автоматично перепідключився
- [ ] Дані почали оновлюватись
- [ ] Немає помилок в консолі

## 5. Логування

### TC-014: Логування в консоль
**Кроки:**
1. Запустити сервер

**Очікуваний результат:**

- [ ] Логи INFO для звичайних операцій
- [ ] Логи WARNING/ERROR для проблем

### TC-015: Перевірка лог файлів

1. Перевірити папку `logs/`

**Очікуваний результат:**
